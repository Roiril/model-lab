# 1サーボ・ロボット設計ガイド（model-lab）

round-bot / square-bot（1サーボ・パン）と周辺部品（servo-test治具・sg92r/horn計測UI・eye・cat-ears/rabbit-ears）を作って得た知見の恒久版。新しいサーボ部品・嵌合部品・アクセサリを作るとき、まずこれを読む。

## 0. アーキテクチャ（最重要）

- **サーボが原点**。先に「サーボ実寸ソリッド」を置き、その周りに胴を生やす。モデルを作ってから穴を空けるのは必ず破綻する（肉厚・配線・ホーン逃げが後から入らない）。
- **機構は `lib/servo_core.py` に一元化**し、各キャラ/治具/アクセサリは import するだけ。外皮（round/square）は皮を差し替える「スキン」に徹する。
- **実物部品の寸法は単一の真実源（config クラス）に集約**：`SERVO`(SG92R) / `HORN` / `EYE`。ここを直せばマウント・受け・全モデルが追従する。
- **計測UIモデル**（`sg92r`, `horn`）＝ mm 入力 + `viewer/preview/*.js` のライブプレビュー。フロー：**実物をノギスで測る → UIに入れる → 形が合う → その値を servo_core の config に焼き込む**（2段階。UIは合わせ込み、config が本番の正）。

## 1. マウント設計の鉄則（ここで何度もハマった）

1. **上から挿入できる形にする。** 胴側に「スラストリング」を付けない——羽根が降りる経路を塞いでサーボが入らなくなる。頭はデッキ上面に直接乗せて回す（頭は軸に固定で回る＝胴とは“結合”しない。だから保持リング不要）。
2. **羽根はデッキ上面のポケットに着座**（重力で安定）。本体はデッキ穴を通って胴内に吊り下がる。羽根（幅広）はデッキ穴を通れないので上で止まる。
3. **頭の内部逃げは「回転対応の丸穴」**。頭は回るので、逃げが角穴だと回した瞬間に当たる。サーボのデッキ上プロファイルを丸で逃がす：
   - 羽根より上の**角型ケース**（数mm）＝**軸からの最遠角までの半径**で丸逃げ（`nub_r = hypot(BODY_L/2 + SHAFT_OFFSET, BODY_W/2)`）。軸はオフセットしているので意外に大きい。
   - **ギアカバー座**（大円柱）＝その径＋クリアランスの丸逃げ。
   - 軸（スプライン）＝中央。
   → この丸逃げを内包できるよう、頭の外形は十分大きく（round-bot は長腕クロスのため半径28に拡大した）。
4. **羽根より上のケースを忘れない。** 羽根の上に角型ケースが数mm突き出ていて、それを逃がさないと頭が浮く。

## 2. ホーン結合（致命的だった点）

5. **クロス受け溝は頭の底（floor）まで縦に貫通**させる。結合面の高さにしか溝が無いと、ホーン腕が溝の下の中実部に当たって**物理的に挿入できない**。腕が下から滑り上がって着座する縦チャネルにする（`cut_horn_coupling(..., floor_z=PLANE)`）。床は底面より少し下げて coplanar を回避。回転キーイングは溝幅で効く。
6. **非対称クロス対応**：長腕・短腕で長さも幅も別（`ARM_SPAN_X/Y`, `ARM_W_X/Y`）。実物は左右非対称なことがある（実測：横35×幅6.8 / たて17×幅3.6）。
7. **結合面の高さ**＝ギアカバー上面 ＋ (小円柱[スプライン]＋ホーンのスタック高 − 腕厚)。スタック高は「ギアカバー上面〜ホーン頂部」を実測（`HORN.STACK_H`）。
8. **ねじ止めの是非**：gentle pan なら**ねじレス（はめ込みキーイング＋摩擦）で十分**。頭は軸固定＋デッキで挟まれて落ちない。ねじ止めするなら**天面ザグリ**で短ねじにする（頭を貫通する長ねじは round で約15mm・square で約21mm＝非現実的）。`cut_horn_coupling(screw=True)` でザグリ＋シャンク穴が出る。

## 3. 配線逃げ

9. サーボの配線は**長手方向の端**から出る（ギアと反対側）。羽受け（フランジ座）にリボンが通る**小窪み**＋胴の底に**下開放スロット**を、**同じ側**に・**内部経由**で設ける。外壁を切るスリットにしない（見栄え＆「回る方向と配線方向が同じ」問題）。向き（±X）はユーザー指定に従う。

## 4. 単位の罠（事故った）

10. **model-lab はメートル単位**（1mm = 0.001）。Blender のメートル・ビューポートでは小物が極小に見えるため、手作業中に**うっかり拡大しがち**（実際カチューシャを59倍に拡大した事故）。実寸化するときは**信頼できる基準寸法**（バンド厚6mm／バンド幅など、耳に影響されない寸法）を測ってそこから均一スケール係数を出す。

## 5. boolean の落とし穴

11. **join（非多様体）後に EXACT boolean → 結果が空になる**。複数プリミティブは穴あけ前に **UNION で多様体化**してから掘る（`add_horn_dummy` / `add_eye_ball` 参照）。
12. カッター面がターゲットと**面一（coplanar）**だと boolean が不安定→**表面より0.5mm程度突き出す**。
13. boolean 後の**STL 法線が内向き**になり Blender workbench では「開いて見える」ことがあるが、メッシュは水密。viewer は `computeVertexNormals()` で再計算するため正しく表示される。**印刷可否は法線でなく水密性で判断**。

## 6. 検証の作法（[[feedback_modellab_visual_check]] と同旨）

14. **自前の Blender ヘッドレス確認レンダはカメラ角度・断面の取り方を間違えて誤判定しがち**（このプロジェクトで何度も別の面＝ドーム上面等を見て「壊れてる」と誤断した）。形の正否は次に寄せる：
    - **数値検証**：bbox 寸法、`bmesh` の non-manifold edge 数、パラメータ計算。
    - **ユーザーの viewer**（http://localhost:3000、法線再計算で正しく表示）での目視。
    - 撮るなら実績アングルのみ（真上 ortho／真横 ortho、look_at を bbox 中心へ）。断面の +Y ortho は空振り多数なので避ける。

## 7. 印刷向けの分割と向き

15. **パーツ分割**：胴／頭は別（間にサーボを挟む）。目は**接着ボール**（くぼみ径と同じ真球＋底を平らに、`add_eye_ball`）。耳はユーザーが Blender で手作り→別印刷。カチューシャはバンド＋耳を統合。
16. **印刷向き（サポート最小）**：胴は上下逆さ（デッキを下）／ドーム・頭はそのまま／アクセサリは面を寝かせて平ら（横置き）／目玉は底平面を下。
17. 出力前に**水密チェック**（non-manifold edge = 0 が理想。1〜数本ならスライサーが自動修復することが多い）。

## 8. Blender 手作業との協調（MCP）

18. ユーザーが Blender で耳等を手作りし、こちらが統合する流れ：
    - band-only を読み込む（**オブジェクトモード必須**。編集モードだと ops の poll が context エラーになる）。
    - シーンのクリアは ops でなく **data API**（`for o in list(bpy.data.objects): bpy.data.objects.remove(o, do_unlink=True)`）が堅い。
    - 統合物は**信頼基準寸法で実寸化** → 横置き回転（厚みを Z へ）→ 中心化 → `bpy.ops.wm.stl_export(... export_selected_objects=True)` で `prints/<name>-assembled.stl` へ。
    - **形・位置はユーザーのを尊重**（こちらはサイズと向きだけ整える）。
    - 手作りジオメトリは .blend にしか残らない → ユーザーに保存を促す。

## 9. パイプライン保守

19. **新モデルを足したら `tools/print_export.py` の `MODELS` に追加**（eye/horn/cat-ears を入れ忘れて prints に出ない事故あり）。サイズ違いが要る部品（目：round⌀6.4/square⌀6.8）は別名で明示書き出し。
20. viewer：`params.py` の `# CATEGORY: 名前` でカテゴリ分け。mm入力モデルは params を mm で書き model.py で `*0.001`。数値入力欄は input 中に値を書き戻さない（書き戻すと小数点が打てなくなる）。

## 10. 定番コマンド

```bash
./run.sh models/<name>/model.py          # 単体ビルド→exports/<name>.stl
node server.js                            # viewer (http://localhost:3000)
blender --background --python tools/print_export.py   # 全パーツを prints/ へ
```
水密チェックや寸法確認は `blender --background --python` の使い捨てスクリプトで `bmesh`/`dimensions` を出す。
