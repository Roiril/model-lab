変更をコミットしてリモートにプッシュしてください。

## 手順

### 1. 変更内容を確認する
```
git status
git diff
```

### 2. コミットメッセージのプレフィックスを選ぶ

| プレフィックス | 用途 |
|----------------|------|
| `feat`     | 新しいモデル・機能の追加 |
| `fix`      | バグ修正・形状の修正 |
| `refactor` | 動作を変えないコード整理 |
| `chore`    | ビルド・設定・依存関係の変更 |
| `docs`     | ドキュメント・コメントのみの変更 |

### 3. コミットメッセージを組み立てる

フォーマット:
```
<prefix>: <何をしたか（日本語）>
```

例:
- `feat: ケーブルクリップモデルを追加`
- `fix: ボックスの壁厚が薄すぎる問題を修正`
- `refactor: params.pyの定数名を統一`

### 4. ステージングしてコミット・プッシュ
```
git add <関連ファイル>
git commit -m "<message>"
git push
```

### 5. 結果を報告する
- コミットハッシュとメッセージを伝える
- プッシュ先のブランチ名を明示する
